# 사용자 페르소나

## 페르소나 1: 고객 (Customer)

### 기본 정보
- **이름**: 김민수
- **나이**: 32세
- **직업**: 회사원
- **역할**: 레스토랑 방문 고객

### 배경
- 주 2-3회 외식을 즐기는 직장인
- 점심시간이 짧아 빠른 주문과 서비스를 선호
- 스마트폰과 태블릿 사용에 익숙함
- 메뉴 선택에 시간이 걸리는 편

### 목표 및 동기
- **주요 목표**: 빠르고 편리하게 음식 주문하기
- **동기**:
  - 직원을 기다리지 않고 즉시 주문하고 싶음
  - 메뉴를 천천히 살펴보고 선택하고 싶음
  - 추가 주문을 부담 없이 하고 싶음
  - 내가 주문한 내역을 확인하고 싶음

### 불만 사항 및 니즈
- **현재 불만**:
  - 직원을 부르기 위해 기다려야 함
  - 메뉴판이 지저분하거나 정보가 부족함
  - 추가 주문 시 다시 직원을 불러야 하는 번거로움
  - 주문한 내역을 잊어버림
- **니즈**:
  - 언제든지 주문할 수 있는 시스템
  - 명확한 메뉴 정보와 이미지
  - 주문 전 장바구니에서 검토 가능
  - 주문 내역 확인 기능

### 기술 수준
- **디지털 리터러시**: 높음
- **모바일 앱 사용**: 능숙함
- **웹 브라우저 사용**: 능숙함
- **새로운 시스템 적응**: 빠름

### 사용 시나리오
1. 레스토랑에 도착하여 테이블에 앉음
2. 테이블 태블릿에서 자동으로 메뉴 화면 확인
3. 카테고리별로 메뉴를 탐색하며 선택
4. 장바구니에 담고 수량 조절
5. 주문 확정 후 음식 대기
6. 추가로 필요한 메뉴가 있으면 다시 주문
7. 식사 후 주문 내역 확인

## 페르소나 2: 매장 운영자 (Store Manager)

### 기본 정보
- **이름**: 박지영
- **나이**: 38세
- **직업**: 레스토랑 매니저
- **역할**: 매장 운영 및 주문 관리

### 배경
- 5년 경력의 레스토랑 매니저
- 10-15개 테이블을 운영하는 중소형 레스토랑 관리
- 피크 시간대 주문 관리에 어려움을 겪음
- 인건비 절감과 운영 효율성 향상에 관심

### 목표 및 동기
- **주요 목표**: 효율적인 주문 관리와 고객 서비스 향상
- **동기**:
  - 주문 누락 및 실수 최소화
  - 실시간으로 모든 테이블의 주문 상황 파악
  - 피크 시간대 주문 처리 속도 향상
  - 인건비 절감 (주문 접수 인력 감소)
  - 매출 및 주문 데이터 관리

### 불만 사항 및 니즈
- **현재 불만**:
  - 수기 주문으로 인한 실수 발생
  - 여러 테이블의 주문을 동시에 관리하기 어려움
  - 주문 내역 추적이 어려움
  - 피크 시간대 혼란스러운 주문 관리
  - 과거 주문 데이터 조회 불편
- **니즈**:
  - 실시간 주문 모니터링 시스템
  - 테이블별 주문 현황 한눈에 파악
  - 주문 상태 관리 (대기중/준비중/완료)
  - 잘못된 주문 삭제 기능
  - 테이블 세션 관리 (고객 입장/퇴장)
  - 과거 주문 내역 조회

### 기술 수준
- **디지털 리터러시**: 중상
- **관리 시스템 사용**: 경험 있음
- **웹 브라우저 사용**: 능숙함
- **새로운 시스템 적응**: 보통 (교육 필요)

### 사용 시나리오
1. 매장 오픈 시 관리자 시스템 로그인
2. 대시보드에서 모든 테이블 상태 모니터링
3. 새로운 주문이 들어오면 실시간 알림 확인
4. 주문 상태를 "준비중"으로 변경
5. 음식 서빙 완료 후 "완료"로 변경
6. 잘못된 주문이 있으면 삭제 처리
7. 고객 퇴장 시 테이블 세션 종료 (매장 이용 완료)
8. 필요 시 과거 주문 내역 조회
9. 메뉴 추가/수정/삭제 관리
